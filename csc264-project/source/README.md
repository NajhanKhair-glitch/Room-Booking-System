## 💳 Booking + Payment Flow

"Book a Space" is the single booking path. The student journey:

```
1. Book a Space            → grid of room cards (photo, capacity, 3-tier price)
2. Click a card            → Space Detail: photo, prices (1hr/4hr/day), RBS contact, additional info
3. Availability            → Reserve Step 1: date/time/event/participants + live Payment Summary + weekly timetable + terms
4. Continue                → Reserve Step 2: confirmation summary + total + BOOK NOW
5. Book Now                → Reserve Step 3: order summary + CONFIRM & PAY
6. Confirm & Pay           → booking row created (Pending) → redirect to ToyyibPay (real gateway)
7. ToyyibPay               → pay via FPX online banking / card on ToyyibPay's secure page
8. Return                  → server verifies the bill → payment marked Paid → receipt shown
9. Moderator reviews       → Approve/reject (separate flow)
```

### Real payment gateway — ToyyibPay

Payment uses **ToyyibPay** (real Malaysian FPX + card gateway). One-time setup:

1. Create an account — sandbox `https://dev.toyyibpay.com` (testing) or `https://toyyibpay.com` (live).
2. Copy your **User Secret Key** and create a **Category** (copy its **Category Code**).
3. Paste both into `php/toyyibpay_config.php`. Until then, the app says the gateway isn't configured (no payment happens).

**Localhost note:** ToyyibPay's server-to-server callback can't reach `localhost`, so on return
the server verifies the bill via `getBillTransactions` (an outbound call that works from localhost).
For a public/production host, also set the callback URL (already wired).

### Read Terms & Conditions

The "Read Terms & Conditions" link on Reserve Step 1 opens
`assets/terms_conditions.pdf` (generated by `tools/gen_terms_pdf.py`).

### Payment scope

FREE spaces (e.g. Surau, rate = RM 0.00) skip the gateway and record a RM 0.00 "Paid" entry.

### Backend endpoints (`payment.php`)

| Endpoint | Purpose |
|----------|---------|
| `POST ?action=quote` | Returns `{rate, hours, days, amount, breakdown}` for a proposed booking |
| `POST ?action=create_bill` | Creates a ToyyibPay bill, returns `{pay_url}` to redirect to (or `{free:true}`) |
| `GET ?action=return` | Browser returns here after paying → verifies bill → marks Paid → redirects to dashboard |
| `POST ?action=callback` | ToyyibPay server-to-server notification → verifies + updates payment |
| `GET ?action=receipt&reservation_id=X` | Returns booking + payment details for the receipt |

## 🛠️ Database Setup

The data (demo accounts + all UiTM Tapah spaces) already lives in the
`bsu_space_booking` database in phpMyAdmin — there is **no seed file to import**.

- `php/database_schema_v2.sql` is kept as the **structure reference** (tables + views)
  for documentation / the ERD requirement.
- To **back up or move to another machine**: phpMyAdmin → select `bsu_space_booking`
  → **Export** → Go. That one file contains the structure *and* the data. Import it
  on the other machine, then open `php/setup.php` to hash the demo passwords.

## 🎨 Design System (warm-neutral)

Every page loads two stylesheets in this order:

1. The page's own CSS (`login_styles.css`, `user_dashboard_styles.css`, etc.)
2. **`bsu_theme.css`** — shared override that flips the entire system to
   the warm-neutral palette and component aesthetic

| Token | Light | Dark |
|-------|-------|------|
| `--bg` | `#f4f2ee` (warm off-white) | `#14110d` |
| `--surface` | `#ffffff` | `#1c1916` |
| `--ink` | `#1c1c1a` (near-black) | `#f4f2ee` |
| `--accent` | `#5F259F` (UiTM purple, used sparingly) | same |
| Headings | Plus Jakarta Sans | same |
| Body | Inter | same |

Buttons use **ink-black primary** that turns UiTM purple on hover. The override
file also contains the **payment modal** component styles (`.pay-*` classes),
so they're available on every page that loads `bsu_theme.css`.

## 🆕 Adding a New Page

1. Create `source/your_new_page.html`
2. Create `css/your_new_page_styles.css`
3. Create `js/your_new_page_script.js`
4. Link them in your HTML `<head>` and at the end of `<body>`
5. Add `<script src="../js/i18n.js"></script>` before your page script for language support
6. Update this README with the new entry
